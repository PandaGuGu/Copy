# 运营后台模块设计规划

> 参考：B站创作中心、YouTube Studio、抖音创作者后台、阿里云控制台  
> 目标：覆盖内容审核、用户服务、风险管控、数据决策、系统运维五大职能

---

## 1. 工单管理 (Ticket System)

### 定位
用户反馈与运营工单的统一入口，类似 B站「客服中心」+ 内部流转。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 工单创建 | 用户提交问题/申诉，支持分类（账号/内容/BUG/建议） | ✅ |
| 工单流转 | 指派 → 处理中 → 待用户确认 → 已解决 | 🟡 缺指派 |
| 内部留言 | 运营人员与用户间站内消息 | ✅ TicketMessage |
| SLA 时效 | 按优先级设超时告警（紧急 2h / 普通 24h） | ❌ 缺失 |
| 快捷回复 | 预设回复模板 | ✅ CSTemplate |
| 自动关闭 | 用户 48h 未回复自动关单 | ❌ 缺失 |

### 行业标准对齐
- B站客服工单：分类 + 优先级 + 自动分配
- 建议增加：**满意度评分**（用户关闭后评分）、**知识库关联**

---

## 2. 风控管理 (Risk Management)

### 定位
内容安全与反垃圾的第一道防线，类似 YouTube Content ID + B站风控中台。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 关键词规则 | 敏感词/广告词正则匹配 | ✅ RiskRule |
| 黑白名单 | IP/UID/设备维度拉黑放行 | ✅ BlackWhiteList |
| 批量处置 | 命中规则后自动：屏蔽/限流/删除 | 🟡 缺动作执行 |
| 风控日志 | 每次命中记录，可追溯 | ❌ 缺失 |
| 行为分析 | 刷量检测（同一IP短时大量播放） | ❌ 缺失 |
| 人工复审 | AI 标疑 → 人工确认的流程 | ❌ 缺失 |

### 行业标准对齐
- B站风控：多维度（设备指纹 + IP + 行为序列）
- 建议增加：**处置申诉入口**（被误判者可申诉）

---

## 3. 版权管理 (Copyright Management)

### 定位
侵权投诉与版权保护，类似 YouTube DMCA + B站「版权保护平台」。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 投诉提交 | 权利人提交侵权证据（URL/截图/权利证明） | ✅ CopyrightComplaint |
| 审核流 | 受理 → 审核 → 通知被诉方 → 处置/驳回 | 🟡 缺通知链路 |
| 反通知 | 被诉方提交抗辩材料 | ❌ 缺失 |
| 版权指纹 | 视频指纹库，自动比对新上传稿件 | ❌ 缺失 |
| 申诉统计 | 按 UP 主/分区统计投诉量 | ❌ 缺失 |

### 行业标准对齐
- YouTube 标准 DMCA 流程：投诉 → 下架 → 反通知 → 恢复
- 建议增加：**7 天反通知等待期**后自动恢复

---

## 4. 数据报表 (BI & Analytics)

### 定位
多维数据看板，支撑运营决策，类似 B站「数据中心」+ Google Analytics。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 实时概览 | 在线人数、今日新增、实时弹幕量 | ✅ 基础 |
| 视频分析 | 播放/互动/留存率/观众画像 | 🟡 缺留存率 |
| 用户分析 | 新老用户比例、来源渠道、活跃时段 | ❌ 缺失 |
| 营收报表 | 硬币/充电/B币流水 | ❌ 缺失 |
| 自定义报表 | 时间范围 + 维度 + 指标自由组合 | ✅ SavedReport |
| 数据导出 | CSV/Excel 导出 | ❌ 缺失 |
| 定时推送 | 日报/周报邮件 | ❌ 缺失 |

### 行业标准对齐
- YouTube Studio Analytics：视频级留存曲线 + 流量来源
- 建议增加：**漏斗分析**（曝光 → 点击 → 观看 → 互动）

---

## 5. 客服后台 (Customer Service)

### 定位
运营人员与用户实时沟通，类似 B站「客服消息」+ 企业微信。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 会话管理 | 用户发起 → 客服接入 → 转接 → 关闭 | ✅ CSConversation |
| 消息收发 | 文本/图片/表情 | ✅ CSMessage |
| 用户画像 | 侧栏展示用户历史记录/投诉记录 | ❌ 缺失 |
| 快捷回复 | 预设模板一键发送 | ✅ CSTemplate |
| 排队系统 | 多客服均分/轮流分配 | ❌ 缺失 |
| 满意度评价 | 关闭会话后用户评分 | ❌ 缺失 |
| 会话质检 | 抽查对话质量、评估客服 KPI | ❌ 缺失 |

### 行业标准对齐
- 在线客服标准：排队 → 分配 → 沟通 → 评价 → 质检闭环
- 建议增加：**敏感词拦截**（客服消息实时过滤脏话/广告）

---

## 6. 运维监控 (Operations Monitor)

### 定位
服务健康度与告警，类似阿里云云监控 + Grafana。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 服务指标 | CPU/内存/磁盘/QPS/响应时间 | ❌ 缺采集 |
| 告警规则 | 阈值触发 → 企微/邮件通知 | ✅ AlertRule |
| 告警记录 | 历史告警 + 处理状态 | ✅ AlertRecord |
| 链路追踪 | 慢请求追踪（OpenTelemetry） | ✅ TraceRecord |
| 定时任务 | 清理/备份/统计 Job 管理 | ✅ TaskLog |
| CDN 刷新 | 批量刷新/预热 | ✅ CDNRefreshTask |
| OSS 生命周期 | 冷热数据自动分层/过期删除 | ✅ OSSLifecycleRule |
| 可视化大屏 | 实时仪表盘 | ❌ 缺失 |

### 行业标准对齐
- 腾讯云可观测平台：指标 + 告警 + 链路一套
- 建议增加：**拨测监控**（从公网定时请求健康检查）

---

## 7. 配置发布 (Config Management)

### 定位
灰度发布与配置管理，类似 Apollo + LaunchDarkly。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| Feature Flag | 功能开关，按用户/比例灰度 | ✅ FeatureFlag |
| 发布记录 | 版本历史 + 回滚 | ✅ ReleaseRecord |
| 审批流 | 发布前需审批 | ❌ 缺失 |
| A/B 实验 | 分组 + 指标对比 | ❌ 缺失 |
| 环境隔离 | dev/staging/prod 独立配置 | ❌ 缺失 |

### 行业标准对齐
- Feature Flag 最佳实践：kill switch + 百分比灰度 + 用户白名单
- 建议增加：**发布日历**（可预约发布时间窗口）

---

## 8. 权限审计 (RBAC & Audit)

### 定位
操作员权限与行为追溯，类似 AWS IAM + 阿里云 RAM。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 角色管理 | 创建角色（超管/审核员/客服/只读） | ✅ AdminRole |
| 权限分配 | 按模块精确控制（菜单级 + 数据级） | ✅ 菜单级 |
| 人员绑定 | 管理员 → 角色，可多角色 | ✅ AdminRoleAssignment |
| 操作审计 | 谁在什么时间做了什么操作 | ✅ AuditLog |
| 登录日志 | IP+时间+设备 | ❌ 缺失 |
| 审批流 | 敏感操作需审批（删视频/封号） | ✅ ApprovalFlow |
| 会话管理 | 强制下线/令牌过期时间 | ❌ 缺失 |

### 行业标准对齐
- IAM 三要素：主体（谁）→ 操作（干什么）→ 资源（哪个视频/用户）
- 建议增加：**操作二次确认**（删视频需输入验证码）、**临时提权**

## 9. 字幕管理 (Subtitle Admin)

### 定位
全站字幕质量管控，创作者上传 + 运营人工修正。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 字幕列表 | 按视频/语言筛选 | ✅ |
| 内容审查 | 查看字幕内容（敏感词检测） | 🟡 缺敏感词 |
| 删除管理 | 违规字幕下架 | ✅ |
| 质量标记 | 标记「机翻/劣质/含广告」 | ❌ 缺失 |
| 字幕审核 | 用户上传后需人工审核才公开 | ❌ 缺失 |
| 贡献者系统 | 显示字幕贡献者、感谢机制 | ❌ 缺失 |

### 行业标准对齐
- B站字幕：众包翻译 + 审核 + 贡献者排行
- 建议增加：**字幕审核队列** + **贡献者积分**

### 定位
运营活动搭建与效果追踪，类似 B站「活动中心」+ 阿里云活动后台。

### 核心功能
| 功能 | 说明 | 当前状态 |
|------|------|---------|
| 专题页 | 聚合视频/文章/动态的内容页 | ✅ SpecialPage |
| 活动管理 | 投稿活动/投票/抽奖 | ✅ Campaign |
| 内容块 | 自定义布局块（Banner/视频列表/文本） | 🟡 JSON 编辑 |
| 参与统计 | 活动参与人数/投稿数/投票数 | ❌ 缺失 |
| 可视化搭建 | 拖拽式页面编辑器 | ❌ 缺失 |
| 定时发布 | 预约活动开始/结束时间 | ✅ 时间字段 |
| 激励发放 | 积分/优惠券/徽章发放 | ❌ 缺失 |

### 行业标准对齐
- B站活动中心：H5 活动页 + 数据看板 + 奖品发放
- 建议增加：**可视化搭建器**（替换 JSON 编辑）、**活动模板库**

## 优先级排序

| 优先级 | 模块 | 原因 |
|--------|------|------|
| P0 | 风控增强（行为分析/处置执行） | 内容安全不可妥协 |
| P0 | 权限审计增强（登录日志/会话管理） | 安全合规基础 |
| P1 | 数据报表增强（导出/漏斗/定时推送） | 运营决策刚需 |
| P1 | 客服后台增强（排队/画像/质检） | 用户体验直接影响 |
| P2 | 版权增强（反通知/指纹库） | 法律合规但需版权素材积累 |
| P2 | 工单增强（SLA/自动关闭/满意度） | 提升效率 |
| P2 | 专题搭建器（可视化） | 降低运营门槛 |
| P3 | 运维大屏 | 锦上添花 |
| P3 | 字幕审核队列 | 内容量小的时候不需要 |

---

## 总结

当前 10 个模块的**骨架**都已搭建完毕（CRUD + 路由 + 前端页面），但大多停留在"能增删改查"的阶段。真正的差距在：

1. **流程闭环** — 大部分模块只有数据的增删改，缺少完整的业务流转（工单 SLA、风控处置执行、版权通知链路）
2. **数据分析** — BI 报表缺少用户画像、流量来源、留存率等核心指标
3. **安全合规** — 权限缺少操作二次确认、登录日志，风控缺行为分析
4. **体验细节** — 可视化搭建、快捷操作、批量处理等提效功能缺失
