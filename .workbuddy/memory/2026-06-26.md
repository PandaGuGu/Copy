## 配置与发布模块：真正的「配置新模块并发布上线」流程（2026-06-26 修正）

### 之前问题
用户反馈：GitHub 推送不是他们要的。他们要的是「在这个模块里配置新模块/新功能开关，然后一键发布上线让配置生效」。

### 修正后的完整流程
1. **功能开关 tab**：新建/编辑/删除 Feature Flag，实时 toggle 开关状态
2. **版本发布 tab**：
   - 「新建发布」→ 自动**快照当前所有 Feature Flag 状态** → 存为 draft
   - 「部署上线」→ 读取快照 → **逐条 apply 到 live DB** → 当前线上版本切换为 deployed
   - 「导出」→ 下载 JSON 快照
   - 「查看快照」→ 弹窗预览所有开关状态

### 具体变更
- `AdminCreateRelease`：创建时自动调用 `buildConfigSnapshot()` 生成快照，状态 `"draft"`
- `AdminDeployRelease` `POST /config/releases/:id/deploy`：解析快照 JSON → 遍历 `feature_flags[]` → 逐条 UPDATE 到 DB（enabled/rollout_pct/whitelist）→ 旧 `deployed` 标记 `rolled_back` → 当前置为 `deployed`
- 前端：按钮从「发布(GitHub)」改为「部署上线」；列表显示部署状态（草稿/已部署/已回退）
- 回滚：`POST /config/releases/:id/rollback` 仍保留

### 状态流转
draft → deployed（部署上线时）
deployed → rolled_back（新版本部署或手动回滚时）

### 删除内容
- 移除 pushToGitHub() 函数及相关 GitHub API 调用
- 移除 bytes/base64/io/os imports
- 移除 .env 中 GitHub 配置注释
- 修正 AdminRollbackRelease：回滚时重新 apply 快照配置 + 状态更新为 deployed/rolled_back

## 字幕管理模块修复 + 表单（2026-06-26 上午）

### 根因
`minibili.ts` 中 `mbAdminListSubtitles` 和 `mbAdminDeleteSubtitle` 使用了普通 `http`（用户 auth），而非 `adminHttp`（admin auth）。后端 API 正常但前端无 admin token → 401 → "加载字幕列表失败"。

### 修复内容
| 文件 | 变更 |
|------|------|
| `minibili.ts` | 导入 `adminHttp`；admin subtitle CRUD 全部改用 `adminHttp` |
| `subtitle.go` | 新增 `AdminCreateSubtitle` `POST /admin/subtitles` + `AdminUpdateSubtitle` `PUT /admin/subtitles/:id` |
| `router.go` | 注册 POST/PUT 路由（同 subtitle:manage 权限组） |
| `SubtitleManage.vue` | 全量重写：新增「新增字幕」按钮、创建/编辑对话框表单（video_id/lang/title/format/content）、编辑按钮 |

### 验证
- `POST /admin/subtitles` → 创建字幕返回 code:0 ✅
- `PUT /admin/subtitles/:id` → 更新字幕 title 返回 code:0 ✅
- `DELETE /admin/subtitles/:id` → 删除返回 code:0 ✅
- Go build 通过，服务重启成功
