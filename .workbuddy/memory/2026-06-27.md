## 直播历史记录加固（GetLiveRoom 双保险）

**问题**：`RecordLiveViewHistory` 仅在 WebSocket `ServeLiveChat` 中调用，但 WebSocket 连接可能失败/未登录时 token 为空，导致直播历史始终不写入。

**修复**：在 `GetLiveRoom` (HTTP API) 中也记录——打开直播间页面时必然调用。
- `internal/handler/live.go` → `GetLiveRoom`：从 Authorization header 提取 Bearer token，解析 JWT 得到 uid，调用 `RecordLiveViewHistory(uid, roomID, "web")`
- http.js interceptor 自动给所有请求加 `Authorization: Bearer <token>`，无需改前端
- `ServeLiveChat` 中的 WebSocket 路径保留作为补充

**双路径保障**：HTTP 页面加载 → WebSocket 连接，两处任一成功即写入历史

## 首页 navMenu 历史悬浮窗 tab 筛选修复

**问题**：顶部导航栏 hover "历史"弹出的悬浮窗有「视频」「直播」「专栏」三个 tab，但切换 tab 只是改了高亮，数据没有任何过滤，所有记录混在一起显示，且点击只跳视频页。

- `historyGrouped` 改用 `historyFiltered` 而非 `historyRawItems`
- `loadHistoryData` 映射新增 `live_room_id`、`article_id` 字段保留
- `goHistoryItem` 重构：直播→`minibiliLiveRoomRoute`，专栏→`minibiliArticleReadRoute`，视频→`minibiliVideoPlayRoute`
- 新增从 minibiliRoutes 导入 `minibiliLiveRoomRoute`、`minibiliArticleReadRoute`、`minibiliVideoPlayRoute`

## 转码封面截帧失败修复

- **问题**：ffmpeg ScreenshotJPEG 对某些像素格式（yuvj420p / full-range YUV）用 mjpeg encoder 失败（"Non full-range YUV is non-standard"），导致转码无限重试
- **修复**：`worker/transcode.go` 封面截帧失败时不再丢弃整个任务，改为生成 1x1 黑色 fallback JPEG（`generateFallbackCoverJPEG`），视频/封面 OSS 上传正常继续
- 非永久性截帧错误：跳过截帧 → 写 fallback cover → 继续后续流程
- 永久性截帧错误（无效数据/不支持编码等）：保持原有 fail 逻辑
