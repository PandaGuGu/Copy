## API 限流中间件

### 实现
`middleware/ratelimit.go` — 滑动窗口限流（Redis INCR + EXPIRE）

### 三层体系
| 层级 | 识别方式 | 默认限额 | 环境变量 |
|------|---------|:--:|------|
| Guest | 客户端 IP | 60 req/60s | `RATE_LIMIT_GUEST_MAX` / `RATE_LIMIT_GUEST_WINDOW` |
| User | JWT user_id | 300 req/60s | `RATE_LIMIT_USER_MAX` / `RATE_LIMIT_USER_WINDOW` |
| Admin | JWT admin_id | 1000 req/60s | `RATE_LIMIT_ADMIN_MAX` / `RATE_LIMIT_ADMIN_WINDOW` |

### 相关改动
- `config.go`: 新增 RateLimit* 7个字段
- `errcode.go`: 新增 CodeRateLimited=42900, msg="请求过于频繁"
- `router.go`: TraceRecordMiddleware 之后、路由之前注册 RateLimiter
- `.env.example`: 新增 RATE_LIMIT_* 配置项

### 特性
- Redis 故障时 fail-open（放行），不阻塞流量
- 自动跳过 /health、/ws/*、/live-hls/*、/uploads/*
- 响应头: X-RateLimit-Limit/Remaining/Reset + Retry-After
- 通过 `RATE_LIMIT_ENABLED=false` 可关闭
