## 风控管理全模块修复（2026-06-29 深夜）

### 诊断发现
风控模块 ScanContentRisk 存在 6 个问题：
1. 🔴 前后端 Category/Action 字段值不一致（前端 content/spam/... vs 后端 keyword/rate_limit/...）
2. 🔴 API 路径不一致（前端 /risk/lists vs 后端 /risk/bw-list）
3. 🟠 黑白名单从未生效（表存在但引擎不查）
4. 🟠 Pattern 只做纯子串匹配，不支持正则（注释说支持 regex）
5. 🟡 auto_ban 不封号（只通知不写 user 表）
6. 🟡 过期名单无清理机制

### 修复清单（6 个文件）

| 文件 | 改动 |
|------|------|
| RiskManage.vue | Category→keyword/rate_limit/device_fingerprint/behavior；Action→reject/quarantine/notify_admin/auto_ban；list_type→blacklist/whitelist；API路径 /risk/lists→/risk/bw-list；toggle→/risk/rules/:id/toggle |
| model/module_extend.go | BlackWhiteList 新增 TargetType 字段；新增 RiskRateCounter 模型 |
| model/model.go | User 新增 BanExpiresAt 字段（Status/BannedAt/BannedReason 已有） |
| data/migrate.go | AutoMigrate 新增 RiskRateCounter |
| handler/admin_risk.go | **引擎重写**：先查黑白名单→正则支持(/pattern/)→rate_limit 频率计数→auto_ban 真正封号→quarantine 支持 comment/article_comment/dynamic_comment；新增 AdminUpdateBWEntry、CleanExpiredBWEntries、matchPattern、checkRateLimit、resolveContentOwner、isWhitelisted、isBlacklisted、banUser |
| handler/router.go | 新增 PUT /risk/bw-list/:id 路由 |

### 引擎执行顺序
1. resolveContentOwner → 查内容作者
2. isWhitelisted → 白名单直接放行
3. isBlacklisted → 黑名单直接 reject
4. 加载 enabled 规则按 priority DESC
5. 逐条匹配：regex(以/包裹)或子串；rate_limit 查 RiskHitLog 窗口计数
6. 命中 → RiskHitLog + 按 Action 执行

## 风控管理前台不显示数据修复 (23:43)

**根因**: `RiskRule` 模型没有 `json` tag，Go 默认序列化为 PascalCase（`ID, Name, Category...`），但前端 Vue 模板全用小写/下划线（`id, name, category, rule_type...`），`<el-table-column prop>` 匹配不到数据，表格只有表头无数据行。

**修复**: `internal/model/module_extend.go` RiskRule 结构体所有字段加 `json:"xxx"` tag：
- `ID → id`, `Name → name`, `Category → category`, `RuleType → rule_type`, `Pattern → pattern`, `Action → action`, `DurationSec → duration_sec`, `Enabled → enabled`, `Priority → priority`, `CreatedAt → created_at`, `UpdatedAt → updated_at`

**验证**: 后端 API 返回字段名从 `["ID","Name","Category","RuleType"...]` 变为 `["id","name","category","rule_type"...]` ✅，与前端 template binding 完全对齐。
