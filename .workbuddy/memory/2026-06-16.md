## 视频上传 nil pointer panic 修复
- **问题**：登录后上传视频返回 500「服务器内部错误」
- **根因**：RabbitMQ 未连接时，`mq`（`*queue.Client` 为 nil）赋值给 `deps.MQ`（`queue.TranscodePublisher` 接口），Go 接口包装了 nil 指针后 `a.MQ != nil` 检查通过，调用 `PublishTranscode` 时 nil deref panic
- **修复**：
  1. `main.go`：`mq` 为 nil 时不赋值给 `deps.MQ`，保持接口 nil
  2. `rabbitmq.go`：`PublishTranscode` 增加 `if c == nil || c.ch == nil` 防御检查
- **验证**：curl POST /api/v1/videos → 201 Created，video 入库（status=processing，RabbitMQ 跳过）
- **提交**：`f67f3d9` "fix: RabbitMQ 未连接时视频上传 nil pointer panic"
- **推送**：PandaGuGu/Copy@v0.0.6（GitHub 暂时连不上，待重试）
