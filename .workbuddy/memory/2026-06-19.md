## 导航栏悬浮面板修复 (commit: e277d69)

### 修复内容
1. **hover事件修复**：5个悬浮面板（个人信息/消息/动态/收藏/历史）全部从 `@mouseover/@mouseout` 改为 `@mouseenter/@mouseleave`。原因：mouseout在鼠标移到子元素时也会触发，导致面板闪烁或消失。mouseenter/mouseleave只在鼠标真正进入/离开目标元素时触发。
2. **historyGrouped方法→计算属性**：原定义在methods中但template中作为computed属性引用，导致Vue返回函数引用而非数组结果。移到computed后正常工作。
3. **http导入路径修正**：navMenu.vue位于 `src/components/navMenu/` 下，到 `src/utils/http.js` 只需 `../../utils/http`（非 `../../../utils/http`）。之前路径错误导致Vite构建失败白屏。
4. **数据加载容错改进**：`_collectLoaded/_dynamicLoaded/_historyLoaded` 标记仅在API成功响应后设置，失败时下次hover可重试。

## 视频网格列裁剪修复 (commit: bfa4df5)

### 根因分析
1. **`.app-body { overflow: hidden }`**（App.vue:622）：裁剪所有水平溢出内容，5列网格右边列被裁剪
2. **DOM结构**：存在两层 `.app-body`：
   - 外层：`#app > .app-body`（overflow容器）
   - 内层：`.app-body > .app-body.bili-wrapper`（内容容器，有1400px/padding/box-sizing规则）

### 修复
1. **App.vue**: `.app-body { overflow: hidden }` → 分离为 `overflow-x: hidden; overflow-y: visible`
   - 新增 `html.chrome-home-top .app-body { overflow: visible !important }` 首页不裁剪

   - `.video-feed-module`: 加 `max-width: 100%; overflow: visible`
   - `.video-feed-grid`: 加 `justify-items: stretch; align-items: start`
   - `.feed-video-card`: 加 `min-width: 0; width: 100%`
   - `.cover-info-bar`: 加 `overflow: hidden`
   - `.cover-meta`: 加 `white-space: nowrap; overflow: hidden; text-overflow: ellipsis`

## 导航栏"动态"/"历史"悬浮窗API端点修复

### 问题
首页顶部导航栏的"动态"和"历史"悬浮窗始终显示"暂无动态"/"暂无历史记录"，无法展示真实数据。

### 根因
前端调用了不存在的API端点：
- "动态"调用 `GET /api/v1/dynamics` → 后端无此路由（404）
- "历史"调用 `GET /api/v1/history` → 后端无此路由（404）

正确的后端端点（均需JWT认证）：
- 动态：`GET /api/v1/users/me/dynamics`（返回 `{items: [{id, title, content, images, created_at, ...}]}`）
- 历史：`GET /api/v1/users/me/view-history`（返回 `{items: [{video_id, title, cover_url, duration_sec, viewed_at, uploader_name, ...}]}`）

### 修复 (navMenu.vue)
1. **`loadDynamicData()`**：
   - 端点：`/api/v1/dynamics` → `/api/v1/users/me/dynamics`
   - 参数：`{limit: 8}` → `{page_size: 6}`
   - 新增 `signIn != 1` 前置检查
   - 新增字段映射：`images[0]→cover/pic`, `created_at→ctime`, 注入当前用户的 `uname/face`

2. **`loadHistoryData()`**：
   - 端点：`/api/v1/history` → `/api/v1/users/me/view-history`
   - 新增 `signIn != 1` 前置检查
   - 新增字段映射：`cover_url→cover`, `duration_sec→duration`, `uploader_name→uploader/author`, `video_id→id`
   - 修复 `_historyLoaded` 仅在成功时设置（失败时下次hover可重试）

## "动态"悬停窗数据为空根因分析

### 现象
`http://localhost:8888/#/minibili/dynamics` 页面有动态，但首页悬停窗显示"暂无动态"。

### 根因
Dynamics页面（`loadFeed()`）展示的是**视频+专栏+动态**三种内容混合流，即使 `user_dynamics` 表为空（0条记录），视频也会被显示。而悬停窗只调用 `/users/me/dynamics`，自然返回空数组。

### 修复 (navMenu.vue `loadDynamicData()`)
改为同时拉取两个数据源：
1. `/api/v1/users/me/dynamics?page_size=4` — 个人动态帖
2. `/api/v1/users/me/videos?page_size=4&status=passed` — 个人已发布视频

合并后按时间倒序排列，取最近6条展示。视频条目描述为"投稿了视频"。
