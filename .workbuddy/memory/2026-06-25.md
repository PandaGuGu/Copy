## 2026-06-25 23:00 — 前后端字段对齐修复

修复审查中发现的 4 个问题：

### P0-1: CDN 刷新字段对齐
- `cdnRefreshReq`: `{type, target}` → `{refresh_type, urls[]}`
- `CDNRefreshTask` 模型: `Type`→`RefreshType`, `Target`→`Urls` (JSON text)
- `AdminListCDNRefreshTasks` 返回时自动 `json.Unmarshal` URLs 为数组
- 前端无需改动（原本就发 `refresh_type` + `urls[]`）

### P0-2: 生命周期路径+字段对齐
- 路由: `/ops/oss/lifecycle` → `/ops/storage/lifecycle-rules`
- `OSSLifecycleRule` 模型: 移除 `Action/Days`，新增 `Name/Bucket/IADays/ArchiveDays/DeleteDays`
- 新增 `AdminUpdateOSSLifecycleRule` PUT handler（支持部分更新含 toggle）
- `ossLifecycleReq` 对应更新字段
- 前端无需改动（原本就调 `/ops/storage/lifecycle-rules` 传上述字段）

### P1-3: 告警级别区分
- `AdminListAlertRecords` level 不再硬编码 `"info"`
- 加载关联 AlertRule，根据 operator 和值偏离比例分级：
  - gt/gte 规则: ≥2x → critical, ≥1.3x → warning, 其余 → info
  - lt/lte 规则: ≤0.5x → critical, ≤0.7x → warning, 其余 → info

### P1-4: 生命周期 PUT 端点
- 路由注册 `PUT /ops/storage/lifecycle-rules/:id`
- 前端 `toggleLifecycle()` / `saveLifecycle(id)` 先前 404，现已可用
