## 修复：Admin 后台侧边栏切换组件不刷新（深度排查）

**问题**：点击"数据报表"后再点击其他左侧管理按钮，URL 变化但右侧组件不刷新。

**根因**：Vue 3 (3.5.13) + Vue Router 4 (4.5.0) + createWebHashHistory 组合下，嵌套 `<router-view>` 在首次 hash 导航后无法响应后续路由变化。经深度排查：
- `:key="$route.fullPath"` 无效
- `$forceUpdate()` 无效
- `beforeRouteUpdate` 无效
- `defineAsyncComponent` + `<component :is>` 无效
- `<keep-alive>` exclude 无效（去掉 keep-alive 也不行）
- 纯 `<a href="#/...">` 标签导航也无效
- `location.hash = '...'` 能改变 URL 但 RouterView 仍不更新

唯一可靠的修复：`location.hash + location.reload()` 强制整页重载。

**修复内容**：
1. `AdminLayout.vue`：侧边栏改用 `<a href="javascript:;" @click="navigate(name)">`，`navigate()` 方法执行 `location.hash = '#/admin/xxx'; location.reload()`
2. `BIReport.vue:349`：修复 `let r; ows = []` → `let rows = []` 语法错误
3. `App.vue`：恢复原始 keep-alive 配置

**代价**：每次侧边栏点击会触发整页刷新（有闪烁），但能保证正确渲染。

## 修复：RBAC 权限审计页面 404 错误

**问题**：跳转权限审计页面后显示 "Request failed with status code 404"

**根因**：前后端 API 不匹配，共 7 处问题：
1. **P0** — 后端缺少 `GET /rbac/admins` 路由（`AdminListAdmins` handler 已实现但未注册）
2. **P0** — 前端调用 `/rbac/approvals`，后端路由为 `/rbac/approval-flows`
3. **P1** — 权限分配 `method: 'PUT'` 应为 `method: 'POST'`
4. **P1** — 角色分配 `method: 'PUT'` 应为 `method: 'POST'`
5. **P2** — 权限载荷不匹配：前端发 `permissions` (code 字符串数组)，后端期望 `permission_ids` (数字 ID 数组)
6. **P2** — 角色权限弹窗取值：`permissionForm.role = d` → 应为 `= d.role`
7. — `AdminListAdmins` 不返回角色信息

**修复**：
- `router.go`：新增 `GET /rbac/admins` + `GET /rbac/roles/:id` 路由
- `admin_rbac.go`：`AdminListAdmins` 增加角色关联查询；`AdminAssignRolePermissions` 支持 `permissions` code 字段
- `RBACManage.vue`：修正 6 处接口调用
