## 热搜榜面板（搜索框聚焦触发）
- 需求：搜索框点击/聚焦时悬浮热搜榜，默认10个条目，2列5行白色按钮，每个按钮最左边是对应的数字
- 后端接口：`GET /api/v1/hot-search?limit=10` 返回 `{code, data:{items:[{rank,title,badge}]}}`
- 后端数据来源：Redis ZSET `search:hot` 通过用户搜索行为自动累积，由 `SearchHot.Record()` 写入
- 前端实现涉及3个文件：
  1. `api/index.js`：新增 `getHotSearchItems(limit)` 函数，调用 `/api/v1/hot-search`
  2. `store/modules/header.js`：新增 `hotSearchItems` state、`SET_HOT_SEARCH_ITEMS` mutation、`setHotSearchItems` action
  3. `components/navMenu/navMenu.vue`：
     - 模板：新增 `.hot-search-panel` 面板，包含标题「热搜榜」和 `.hot-search-grid` 2列网格
     - 每个条目：左侧数字排名（`.hot-search-rank`，前三名 `.top3` 标红）、中间标题、右侧 badge（如「热」「新」）
     - 逻辑：`hotSearchVisible` data 控制显示；`hotSearchPanelShow` 计算属性（无输入 + 聚焦 + 无建议面板时显示）
     - 面板优先级：输入建议 > 热搜榜 > 搜索历史
     - 聚焦时调用 `this.$store.dispatch("header/setHotSearchItems")` 获取最新热搜数据
     - 点击热搜条目调用 `searchByHistory(item.title)` 执行搜索
     - CSS：`.hot-search-grid` 使用 `grid-template-columns: 1fr 1fr` 实现2列布局
- 测试：通过 curl 多次调用 `GET /api/v1/search?keyword=...` 触发后端 `SearchHot.Record()` 填充 Redis ZSET
